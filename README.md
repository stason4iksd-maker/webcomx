# ШЁПОТ — сайт веб-комикса

Статический сайт для читалки веб-комикса (трилогия про Ренкаро/Нактира),
рассчитан на хостинг GitHub Pages без своего сервера. Заливка страниц
комикса и управление главами делается прямо из браузера, через
`admin.html`, который обращается к GitHub API.

## Структура

```
index.html          — главная страница (список книг и глав)
reader.html          — читалка конкретной главы (?ch=ID)
admin.html           — админка: создание глав, выбор стиля, заливка страниц
css/main.css          — общие стили сайта
css/themes.css         — фоновые пресеты для читалки (nactir / wasteland / alteria / свой цвет)
css/admin.css          — стили админки
js/data.js            — загрузка data/comics.json
js/app.js             — рендер главной страницы
js/reader.js           — рендер читалки
js/admin.js            — вся логика админки
js/github-api.js        — обёртка над GitHub Contents API
data/comics.json        — все книги, главы, стили и списки страниц
assets/pages/<book>/<chapter>/  — сюда админка складывает картинки страниц
```

## 1. Публикация на GitHub Pages

1. Создайте новый репозиторий на GitHub (публичный или приватный — Pages
   работает с обоими на плане Pro/Team; для личного аккаунта на приватных
   репо Pages тоже доступен).
2. Залейте туда содержимое этой папки целиком (включая `.nojekyll`).
3. В репозитории: **Settings → Pages → Source → Deploy from a branch**,
   выберите branch (например `main`) и папку `/ (root)`.
4. Через минуту сайт будет доступен по адресу вида
   `https://<ваш-логин>.github.io/<репозиторий>/`.

## 2. Токен для админки

Админка сама коммитит файлы в репозиторий через GitHub API, поэтому ей
нужен токен с правом записи:

1. GitHub → **Settings → Developer settings → Fine-grained tokens → Generate new token**.
2. Repository access: **Only select repositories** → выбрать этот репозиторий.
3. Permissions → **Contents: Read and write**.
4. Скопируйте токен (он показывается один раз).

Токен хранится только в localStorage вашего браузера и отправляется
только на `api.github.com`. Если сидите с чужого/общего компьютера —
после работы нажмите «Забыть токен» в админке.

## 3. Как пользоваться админкой

1. Откройте `admin.html` (локально или уже на опубликованном сайте).
2. Впишите Owner / Repo / Branch / Token → «Сохранить настройки».
3. «Загрузить главы из GitHub» — подтянет текущий `data/comics.json`.
4. Дальше можно:
   - добавлять книги и главы;
   - каждой главе назначать **готовую тему** (Нактир / Пепел / Альтерия)
     или **свой цвет фона** — простая заливка без картинки-фона;
   - заливать страницы главы прямо файлами с компьютера (кнопка
     «Загрузить страницы») — они уйдут в
     `assets/pages/<book-id>/<chapter-id>/` и сразу попадут в список
     страниц главы;
   - менять порядок страниц стрелками, удалять страницы (с опцией
     удалить и сам файл из репозитория).
5. Изменения сохраняются в `data/comics.json` автоматически после
   каждого действия (коммитом в репозиторий), плюс есть кнопка
   «Сохранить в GitHub» для ручного форс-сохранения.
6. Через 30–60 секунд после коммита обновлённая версия появится на
   опубликованном сайте (GitHub Pages пересобирается автоматически).

## 4. Локальный просмотр без публикации

Так как страницы используют `fetch('data/comics.json')`, открывать
`index.html` напрямую через `file://` браузер не даст (CORS). Проще
всего поднять локальный сервер в папке проекта:

```bash
python3 -m http.server 8080
```

и открыть `http://localhost:8080/`. Админка при этом всё равно
обращается к настоящему GitHub-репозиторию (это единственный способ,
которым здесь вообще что-то сохраняется).

## 5. Три темы «из коробки»

| Тема       | Для чего                          | Акцент  |
|------------|------------------------------------|---------|
| `nactir`   | Книга I — космос, зал воспоминаний | золотой |
| `wasteland`| Книга II — постапокалипсис, пепел  | ржавый  |
| `alteria`  | Книга III — стеклянная утопия      | ледяной |

Их можно расширять — добавьте новый ключ в `data.themes` внутри
`data/comics.json` (поля `bg`, `bg2`, `accent`, `text`, `label`) и он
сразу появится в выпадающем списке админки. Для полностью нового визуала
стоит также добавить `body.theme-<key>` блок в `css/themes.css` —
иначе такая тема просто будет плоской заливкой цветом `bg`.
